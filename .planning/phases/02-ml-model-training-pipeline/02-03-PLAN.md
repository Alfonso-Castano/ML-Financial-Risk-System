<task type="auto">
  <name>Task 2: Implement training pipeline in train.py</name>
  <files>backend/ml/train.py</files>
  <action>
Create `backend/ml/train.py` with the complete training pipeline. Structure:

**Imports:**
- `json`, `pathlib.Path`
- `numpy`, `pandas`, `torch`, `torch.nn`, `torch.optim`
- `torch.utils.data.DataLoader`
- `sklearn.model_selection.train_test_split`
- `sklearn.preprocessing.StandardScaler`
- `tqdm` (for progress bar during training)
- `backend.config.settings` (all training constants)
- `backend.data.feature_engineering` (build_feature_matrix, FEATURE_NAMES)
- `backend.ml.model` (FinancialRiskModel)
- `backend.ml.dataset` (FinancialDataset)

**Function: `train_epoch(model, loader, criterion, optimizer) -> float`**
- Sets `model.train()`
- Iterates over batches: zero_grad -> forward (with `.squeeze()`) -> loss -> backward -> step
- Returns average loss for the epoch

**Function: `validate_epoch(model, loader, criterion) -> float`**
- Sets `model.eval()`
- Uses `torch.no_grad()` context
- Iterates over batches: forward (with `.squeeze()`) -> loss
- Returns average loss for the epoch

**Function: `save_scaler_stats(scaler: StandardScaler, feature_names: list, path: str) -> None`**
- Extracts `scaler.mean_` and `scaler.scale_` as Python lists (via `.tolist()`)
- Saves JSON: `{"feature_names": [...], "mean": [...], "scale": [...]}`
- Creates parent directory if needed with `Path(path).parent.mkdir(parents=True, exist_ok=True)`

**Function: `main() -> dict`**
The main training function. Steps:

1. **Set random seed:** `torch.manual_seed(42)` and `np.random.seed(42)` for reproducibility
2. **Load data:** `pd.read_csv(settings.DATA_PATH)`
3. **Build feature matrix:** Call `build_feature_matrix(df)` -> `(X, y, profile_ids)`
4. **Split by profile ID:**
   - Use `train_test_split` with `stratify=y` to carve out test set (15%)
   - Split remaining into train/val using `test_size=0.176` (15% of total) with `stratify`
   - Use indices to split X and y arrays (not profile_ids as separate arrays)
   - Specifically: first get indices array `np.arange(len(y))`, split indices, then use indices to slice X and y
5. **Scale features:**
   - Fit `StandardScaler` on `X_train` only
   - Transform `X_train`, `X_val`, `X_test` using the fitted scaler
   - Save scaler stats via `save_scaler_stats()`
6. **Create datasets and loaders:**
   - `FinancialDataset` for train, val, test
   - `DataLoader` with `batch_size` from settings, `shuffle=True` for train, `False` for val/test
   - `num_workers=0` (Windows compatibility)
7. **Initialize model, loss, optimizer:**
   - `FinancialRiskModel(input_size=settings.INPUT_SIZE)`
   - `nn.BCELoss()`
   - `torch.optim.Adam(model.parameters(), lr=settings.LEARNING_RATE)`
8. **Training loop (with tqdm progress bar — per locked decision, no per-epoch print spam):**
   - Wrap epoch range with `tqdm(range(NUM_EPOCHS), desc="Training")` for a progress bar
   - For each epoch:
     - Call `train_epoch()` and `validate_epoch()`
     - Append losses to `train_losses` and `val_losses` lists
     - Update tqdm postfix with current train/val loss: `pbar.set_postfix(train_loss=f"{train_loss:.4f}", val_loss=f"{val_loss:.4f}")`
     - If val_loss improves: save model with `torch.save(model.state_dict(), settings.MODEL_PATH)`, reset patience counter
     - If val_loss doesn't improve for EARLY_STOPPING_PATIENCE epochs: stop early (break out of loop)
9. **Load best model** (the saved checkpoint, not the final epoch model)
10. **Return dict** with: model, X_test, y_test, train_losses, val_losses, num_epochs_trained, split sizes

**`if __name__ == '__main__':` block:**
- Calls `main()`
- Does NOT import or call evaluate.py (per locked decision: evaluate.py is a standalone script)
- Saves training history for standalone evaluate.py to use:
  - Write `models/training_history.json` with `{"train_losses": [...], "val_losses": [...]}` so evaluate.py can generate loss curve plots independently
- Prints a single final summary line after training completes:
  - Model save path
  - Split sizes (train/val/test)
  - Number of epochs trained
  - Best validation loss
  - Example: `"Training complete. Model saved to models/latest_model.pth | Epochs: 42/75 | Best val loss: 0.3214 | Splits: 2101/449/450"`

**Critical implementation details:**
- `.squeeze()` on model output in both train_epoch and validate_epoch
- `model.train()` before training, `model.eval()` before validation
- `torch.no_grad()` during validation
- Create `models/` directory before saving: `Path(settings.MODEL_PATH).parent.mkdir(parents=True, exist_ok=True)`
- Split by shuffled indices with stratification, NOT by profile_id arrays directly
