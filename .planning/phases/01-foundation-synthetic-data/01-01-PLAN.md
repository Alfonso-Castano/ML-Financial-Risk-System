  <action>
    Create `backend/data/synthetic_generator.py` that generates realistic synthetic financial profiles and saves them as a CSV.

    **Generator structure:**

    1. `generate_profiles(num_profiles, seed)` — main entry point:
       - Creates `rng = np.random.default_rng(seed)`
       - Allocates profiles across archetypes based on weights (e.g., 20% struggling, 25% getting_by, etc.)
       - For each profile, calls `_generate_single_profile(rng, archetype_config, profile_id)`
       - Collects all rows into a single DataFrame
       - Applies stress labels
       - Returns DataFrame

    2. `_generate_single_profile(rng, archetype, profile_id)` — generates one profile's 12 months:
       - Generate annual income using `rng.uniform(low, high)` within archetype's income_range (NOT log-normal — archetypes already segment income, so uniform within band is fine)
       - Monthly income = annual / 12
       - Generate credit score: `rng.normal(archetype.credit_score_mean, 40).clip(300, 850).astype(int)`
       - Generate base expense ratio from archetype's expense_ratio range
       - Split expenses into: essentials (~60% of total expenses), discretionary (~20%), debt_payment (~20%)
       - Generate initial savings: `monthly_expenses * rng.uniform(archetype.savings_months[0], archetype.savings_months[1])`
       - For each of 12 months:
         - Apply monthly variance to income: `monthly_income * rng.normal(1.0, MONTHLY_VARIANCE)`
         - Apply monthly variance to essentials and discretionary (debt_payment stays fixed)
         - Calculate total_expenses = essentials + discretionary + debt_payment
         - Calculate monthly cash flow = income - total_expenses
         - Update savings: savings = max(0, previous_savings + cash_flow)
         - Record row: profile_id, month (1-12), income, essentials, discretionary, debt_payment, total_expenses, savings, debt (annual debt = debt_payment * 12), credit_score
       - Return list of 12 row dicts

    3. `_apply_stress_labels(df)` — applies stress rules per profile:
       - Group by profile_id
       - For each profile:
         - Condition 1: final month savings < final month total_expenses
         - Condition 2: max consecutive months with negative cash flow (income - total_expenses < 0) >= 3
         - is_stressed = 1 if either condition is true, else 0
       - Assign is_stressed to all rows of that profile (same label for all 12 months)
       - Return DataFrame with is_stressed column

    4. `save_dataset(df, output_path)` — saves to CSV:
       - Creates parent directory with pathlib
       - Saves with `index=False`
       - Prints summary: row count, profile count, stress ratio, column names

    5. `if __name__ == "__main__":` block:
       - Imports settings
       - Calls `generate_profiles(NUM_PROFILES, RANDOM_SEED)`
       - Calls `save_dataset(df, OUTPUT_PATH)`
       - Prints class balance and basic statistics

    **CSV columns (long format, one row per person-month):**
    `profile_id, month, income, essentials, discretionary, debt_payment, total_expenses, savings, debt, credit_score, is_stressed`

    **Key implementation details:**
    - Use `from backend.config.settings import ...` for all configuration
    - Savings tracks cumulatively — each month updates based on previous month + cash flow
    - Credit score is constant per profile (doesn't change monthly)
    - Debt = debt_payment * 12 (annualized, constant per profile)
    - Include ~10-15% edge cases naturally (archetype overlap near boundaries produces these)
    - Round financial values to 2 decimal places
    - Target ~35% stressed class balance (achieved through archetype design — struggling and getting_by profiles are more likely to trigger stress conditions)

    **After creating the generator, run it:**
    ```bash
    cd project_root && python -m backend.data.synthetic_generator
    ```
