<task type="auto">
  <name>Task 1: Create Pydantic schemas for predict and health endpoints</name>
  <files>backend/api/schemas.py</files>
  <action>
Create Pydantic v2 models in `backend/api/schemas.py`:

**MonthlyEntry** (nested model for each month):
- `income: float` (required)
- `expenses: float` (required)
- `debt_payment: float = 0.0` (optional, defaults to 0)
- `@field_validator('income', 'expenses')` — reject negative values with `ValueError(f"{info.field_name} must be non-negative")`
- `@field_validator('debt_payment')` — reject negative values

**PredictRequest** (top-level request body):
- `months: list[MonthlyEntry]` (6-12 entries)
- `credit_score: float` (required)
- `@model_validator(mode='after')` — validate `6 <= len(self.months) <= 12`, raise ValueError if not
- `@field_validator('credit_score')` — must be between 300 and 850

**ComputedFeatures** (returned in response):
- One field per FEATURE_NAMES entry: `avg_income`, `avg_expenses`, `final_savings`, `debt_payment`, `credit_score`, `debt_ratio`, `liquidity_ratio`, `net_cash_flow`, `consec_negative_months` — all `float`

**InsightsObject**:
- `risk_factors: list[str]` — plain-language risk factor sentences
- `summary: str` — one paragraph, financial-advice tone

**PredictResponse**:
- `risk_score: float` (0-100)
- `risk_category: str` ("low", "medium", or "high")
- `probability: float` (0-1, raw model output)
- `insights: InsightsObject`
- `computed_features: ComputedFeatures`
- `debt_payment_defaulted: bool` (True if any month's debt_payment used the default 0)

**HealthResponse**:
- `status: str` ("healthy")
- `model_loaded: bool`
- `feature_count: int`
- `metrics: dict` (recall, roc_auc from metrics.json)

Use Pydantic v2 imports: `from pydantic import BaseModel, field_validator, model_validator`.

<task type="auto">
  <name>Task 2: Create Predictor class with full inference pipeline</name>
  <files>backend/ml/predictor.py</files>
  <action>
Create `backend/ml/predictor.py` with a `Predictor` class that orchestrates the entire inference flow.

**`__init__(self)`:**
- Load model: `FinancialRiskModel(input_size=settings.INPUT_SIZE)`, `torch.load(settings.MODEL_PATH, weights_only=True)`, `model.load_state_dict()`, `model.eval()`
- Load scaler: read `settings.SCALER_PATH` as JSON, extract `mean` and `scale` into `np.array(..., dtype=np.float32)`
- Load metrics: read `settings.METRICS_PATH` as JSON, store as `self.metrics`
- If any file is missing, raise `RuntimeError` with a clear message (fail-fast pattern)

**`predict(self, request: PredictRequest) -> dict`:**
1. Call `self._build_dataframe(request)` to construct a pandas DataFrame
2. Call `engineer_features(profile_df)` from `backend.data.feature_engineering`
3. Assemble feature vector: `np.array([features[name] for name in FEATURE_NAMES], dtype=np.float32)`
4. Z-score scale: `scaled = (feature_vector - self.mean) / self.scale`
5. Inference: `with torch.no_grad():` → `torch.tensor(scaled).unsqueeze(0)` → `float(self.model(tensor).item())`
6. Derive: `risk_score = round(prob * 100, 1)`, `risk_category` using thresholds (>=0.65 high, >=0.35 medium, else low)
7. Call `self._compute_insights(features, prob)` for insights
8. Check `debt_payment_defaulted`: True if ANY month in request has debt_payment exactly equal to the default 0.0 value AND the original request JSON didn't explicitly set it. Simplify: check if any `month.debt_payment == 0.0` (since the Pydantic default is 0.0, this catches both explicit zero and omitted).
9. Return dict matching PredictResponse schema fields

**`health(self) -> dict`:**
- Return `{"status": "healthy", "model_loaded": True, "feature_count": len(FEATURE_NAMES), "metrics": {"recall": self.metrics["recall"], "roc_auc": self.metrics["roc_auc"]}}`

**`_build_dataframe(self, request) -> pd.DataFrame`:**
- Map API fields to training column names (critical — see research Pitfall 1):
  - `month.income` → `income`
  - `month.expenses` → `total_expenses` (NOT `expenses`)
  - Compute running `cumulative_savings = max(0, cumulative_savings + income - expenses - debt_payment)` for each month
  - `month.debt_payment` → `debt_payment`
  - `request.credit_score` → `credit_score` (same for every row)
  - `i + 1` → `month` column
- Return `pd.DataFrame(rows)`

**`_compute_insights(self, features: dict, prob: float) -> dict`:**
- Build `risk_factors` list of plain-language sentences. Check ALL of the following (include every one that triggers — not capped):
  - `liquidity_ratio < 1.0`: "Low emergency fund: savings cover only {X} months of expenses (recommended: 1+ months)"
  - `consec_negative_months >= 3`: "Sustained cash flow deficit: {N} consecutive months of negative cash flow"
  - `debt_ratio > 0.5`: "High debt burden: debt payments consume {X}% of available cash flow after expenses"
  - `net_cash_flow < 0`: "Negative average cash flow: spending exceeds income by ${X}/month on average"
  - `credit_score < 640`: "Credit score of {X} indicates elevated credit risk"
- Build `summary` string: financial-advice tone, not technical. Vary by risk level:
  - No risk factors: "Your finances appear healthy. Maintain your savings buffer and continue managing expenses within income."
  - High risk (prob >= 0.65): mention priority actions based on top factors
  - Medium/some factors: "Some financial risk indicators present. Monitor your cash flow closely and build your emergency fund."
- Return `{"risk_factors": risk_factors, "summary": summary}`
