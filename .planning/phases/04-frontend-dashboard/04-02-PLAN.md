<task type="auto">
  <name>Task 1: Implement complete dashboard JavaScript</name>
  <files>frontend/app.js</files>
  <action>
Create frontend/app.js with all dashboard behavior. Structure the file in clear sections with comments. Use textContent (not innerHTML) for all API data. Cache DOM references at initialization.

**Section 1: DOM References & Constants**
- Cache all frequently used elements at top: monthGrid tbody, creditScore input, predictBtn, resultsPanel, validationBanner, csvUpload, insightsContainer, featuresGrid, gauge elements, error elements, model metrics container, etc.
- Define constants: CIRCUMFERENCE = 251.3, ARC = 188.5 (for gauge math)
- Define FEATURE_LABELS map: { avg_income: "Avg Monthly Income", avg_expenses: "Avg Monthly Expenses", final_savings: "Final Savings", debt_payment: "Monthly Debt Payment", credit_score: "Credit Score", debt_ratio: "Debt Ratio", liquidity_ratio: "Liquidity Ratio", net_cash_flow: "Net Cash Flow", consec_negative_months: "Consecutive Negative Months" }

**Section 2: Tab Switching**
- querySelectorAll('.tab-btn'), add click listeners
- On click: remove .active from all .tab-btn and .tab-panel, add .active to clicked btn and corresponding panel (getElementById('tab-' + btn.dataset.tab))

**Section 3: Results Panel State Management**
- showState(state) function: sets one of 'is-empty', 'is-loading', 'has-results', 'is-error' on resultsPanel
- Implementation: remove all state classes, add the specified one
- OR: show/hide state-* child divs by toggling display/hidden
- Initial state on page load: 'is-empty'

**Section 4: Form Validation**
- validateForm() returns { valid: boolean, errors: string[], payload: object|null }
- clearValidationUI(): remove all .field-error classes, remove .error-text elements, hide validationBanner
- getActiveRows(): iterate grid rows from month 1, a row is "active" if income AND expenses inputs are non-empty (not NaN after parseFloat). Stop at first row where both are empty. Return array of row data objects
- Validation rules (from schemas.py):
  - At least 6 active rows ("Enter at least 6 months of data")
  - No more than 12 active rows
  - Each active row: income >= 0, expenses >= 0
  - debt_payment: if filled, must be >= 0 (empty = defaults to 0)
  - credit_score: required, 300-850
  - Check for gaps: if a row after the first empty row has data, error ("No gaps allowed — fill months consecutively from Month 1")
- markFieldError(inputEl, message): add .field-error class to input, insert .error-text span after it
- showValidationBanner(errors): populate validationBanner with error list, make visible

**Section 5: Build Payload**
- buildPayload(activeRows, creditScore): constructs { months: [...], credit_score } matching API contract
- Each month: { income, expenses, debt_payment } (debt_payment defaults to 0.0 if empty)

**Section 6: API Integration (POST /predict)**
- async submitPrediction(payload):
  - Call showState('is-loading')
  - fetch('/predict', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify(payload) })
  - If !res.ok: parse JSON, call showApiError(data.error || `Server error (${res.status})`)
  - If ok: parse JSON, call renderResults(data)
  - Catch: showApiError('Could not reach the server. Is the backend running?')
- Wire predictBtn click handler: call clearValidationUI(), run validateForm(), if invalid show errors and return, else submitPrediction(payload)

**Section 7: Render Results**
- renderResults(data): takes full API response object
  - Call showState('has-results')
  - Call setGauge(data.risk_score) to animate the gauge
  - Set risk category label text (data.risk_category) and probability text
  - Handle debt_payment_defaulted: if true, show debtWarning note ("Some months have no debt payment recorded. If you have debt payments, enter them for a more accurate assessment."); if false, hide it
  - Call renderInsights(data.insights)
  - Call renderFeatures(data.computed_features)

**Section 8: SVG Gauge**
- setGauge(riskScore): riskScore is 0-100
  - Calculate filled = (riskScore / 100) * ARC
  - Set gauge-fill stroke-dasharray to `${filled} ${CIRCUMFERENCE}`
  - Set gauge-fill stroke color based on risk_score: >= 65 → --color-risk-high, >= 35 → --color-risk-medium, < 35 → --color-risk-low (use getComputedStyle to read CSS var, or set stroke directly with hex)
  - Update gauge label text to show the score (e.g., "73")
  - The CSS transition on stroke-dasharray handles the animation automatically

**Section 9: Render Insights**
- renderInsights(insights):
  - Set insightsSummary textContent to insights.summary
  - Clear insightsContainer
  - For each risk_factor in insights.risk_factors:
    - Create insight-card div with data-expanded="false"
    - Create insight-header button with title span (extract short title before ":") and chevron span
    - Create insight-body div with paragraph (full text after ":")
    - Add click listener on header to toggle data-expanded
    - Append to insightsContainer
  - Use textContent for all API strings (XSS prevention)

**Section 10: Render Computed Features**
- renderFeatures(features):
  - Clear featuresGrid
  - For each key in FEATURE_LABELS:
    - Get value from features[key]
    - Format: debt_ratio/liquidity_ratio → toFixed(2), consec_negative_months → Math.round(), dollar values → "$" + Math.round().toLocaleString()
    - Create feature-item div with label span and value span
    - Use textContent for both label and value
    - Append to featuresGrid

**Section 11: Error State**
- showApiError(message):
  - Call showState('is-error')
  - Set errorMessage textContent to message
- Wire retryBtn click: call predictBtn.click() (re-submit with current form data)

**Section 12: CSV Upload**
- csvUpload change listener:
  - Read file with FileReader.readAsText()
  - In onload: parse CSV text with parseCSV(text)
  - parseCSV: split by newlines, parse header row, map data rows to objects
  - csvRowsToPayload: extract months[] and credit_score from parsed rows
  - Populate grid inputs from parsed data (set .value on each input)
  - Set creditScore input value
  - Optionally auto-submit or let user click Predict
- downloadTemplate click handler:
  - Generate CSV string: "month,income,expenses,debt_payment,credit_score\n1,,,,\n2,,,,\n..." (12 rows with empty values)
  - Create Blob, URL.createObjectURL, create temporary anchor, click it, revoke URL
  - Filename: "financial_data_template.csv"

**Section 13: About the Model — Fetch /health**
- On DOMContentLoaded: fetch('/health'), if ok, populate modelMetrics container with recall and roc_auc values using textContent
- If fetch fails, show "Could not load model metrics" in the metrics container
- This runs independently of the Predict tab

**Section 14: Initialization**
- DOMContentLoaded listener wrapping all setup:
  - Cache DOM refs
  - Set initial state to 'is-empty'
  - Wire tab listeners
  - Wire predictBtn listener
  - Wire csvUpload listener
  - Wire downloadTemplate listener
  - Wire retryBtn listener
  - Fetch /health for About tab

Important patterns from research:
- Use textContent for ALL API data (never innerHTML with variable data)
- debt_payment defaults to 0.0 when empty (not NaN)
- Active rows = contiguous from month 1, stop at first row with empty income AND expenses
- stroke-dasharray offset for gauge-bg: "188.5 251.3" with dashoffset "-31.4" (per research)
- Risk score thresholds: high >= 65, medium >= 35, low < 35
