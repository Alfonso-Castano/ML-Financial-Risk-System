IMPORTANT: Before writing any CSS, invoke the `frontend-design` skill to establish the design system. The skill will guide aesthetic choices within the Anthropic-inspired direction locked in CONTEXT.md. Feed it these constraints:
- Anthropic/Claude-inspired aesthetic: warm earth tones (background #eeece2, text #3d3929, accent #da7756, button #bd5d3a)
- Plus Jakarta Sans from Google Fonts as primary typeface
- CSS custom properties for all tokens
- Desktop-first with mobile fallback at 768px

After the skill provides design direction, create frontend/styles.css with:

**CSS Variables (:root):**
- Color system: --color-bg (#eeece2), --color-surface (#f5f3eb), --color-text (#3d3929), --color-text-muted (#6b6654), --color-accent (#da7756), --color-accent-dark (#bd5d3a), --color-border (#d9d6c8), --color-error (#c0392b)
- Risk colors: --color-risk-low (#4caf50), --color-risk-medium (#ff9800), --color-risk-high (#e53935)
- Typography: --font-sans ('Plus Jakarta Sans', ui-sans-serif, system-ui, sans-serif), --font-mono
- Type scale: --text-xs through --text-3xl
- Spacing scale: --space-1 through --space-12
- Border radii: --radius-sm (4px), --radius-md (8px), --radius-lg (12px)

**Layout:**
- Page container with max-width, centered
- Tab navigation bar (.tab-nav, .tab-btn with active state)
- Tab panels (.tab-panel, show/hide with .active class)
- Side-by-side layout for Predict tab: form on left (~55%), results on right (~45%). Use CSS Grid or Flexbox
- Stack vertically below 768px

**Components:**
- Spreadsheet grid table (.month-grid): styled table with header row, 12 input rows. Rows 7-12 visually indicated as optional (reduced opacity or subtle label). Input fields styled consistently with placeholder hints
- Credit score field (.credit-score-row): standalone input below the grid
- CSV upload area: file input with styled drop zone or button + "Download Template" link
- Predict button: prominent, --color-accent-dark background, white text, hover state
- Validation styles: .field-error (red border on input), .error-text (red helper text below field), .validation-banner (summary banner above form with error list)

**Results Panel States (all CSS-driven):**
- .is-empty: friendly placeholder with icon/message ("Enter financial data and click Predict")
- .is-loading: skeleton shimmer placeholders (gauge skeleton, text skeletons, card skeletons). Shimmer uses CSS @keyframes with linear-gradient background sweep
- .has-results: shows gauge, insights, features sections
- .is-error: error message with retry button

**SVG Gauge styles:**
- .gauge: transform rotate(-225deg) for semicircle starting at bottom-left
- .gauge-bg: stroke var(--color-border)
- .gauge-fill: transition on stroke-dasharray (0.8s ease-out) and stroke (0.4s ease)
- .gauge-label: centered text (not rotated — use counter-rotation or separate element)

**Insight cards:**
- .insight-card: card with border, radius
- .insight-header: clickable button with title + chevron
- .insight-body: max-height 0 with overflow hidden, transition max-height 0.3s
- .insight-card[data-expanded="true"] .insight-body: max-height 200px with padding
- .insight-chevron: rotate(180deg) transition when expanded

**Computed features grid:**
- .features-grid: CSS grid (2-3 columns) showing label-value pairs
- .feature-item, .feature-label, .feature-value styles

**About the Model tab:**
- Clean typographic layout for explanatory content
- Styled sections for architecture, features, metrics

**Debt payment defaulted note:**
- .debt-warning: subtle informational note style (not error-colored)

Let the frontend-design skill influence the aesthetic execution (spatial composition, background textures, hover states, micro-details) while honoring the locked color palette and typography choice.

**Body structure:**

2. **Tab navigation:** (role="tablist")
   - "Predict" tab button (active by default, data-tab="predict")
   - "About the Model" tab button (data-tab="about")

3. **Predict tab panel** (id="tab-predict", class="tab-panel active"):

   Left column (form area):
   - Validation summary banner container (id="validationBanner", hidden by default)
   - Spreadsheet grid table (id="monthGrid"):
     - thead with columns: Month, Income ($), Expenses ($), Debt Payment ($)
     - tbody with 12 rows, each row has data-month="1" through data-month="12"
     - Each row: month label td + 3 input tds
     - Inputs: type="number", class="grid-input income/expenses/debt", placeholder hints (income: "4500", expenses: "3200", debt: "500"), min="0", step="0.01"
     - Rows 7-12 should have a CSS class like "optional-row" for visual distinction
     - After row 6, optionally add a visual separator or label indicating "Optional"
   - Credit score row below grid: label + input (id="creditScore", placeholder="720", min="300", max="850", step="1")
   - CSV upload section: file input (id="csvUpload", accept=".csv") + "Download CSV Template" link/button (id="downloadTemplate")
   - Submit button: "Predict Risk" (id="predictBtn")
   - Form should have novalidate attribute to suppress native validation

   Right column (results area, id="resultsPanel"):
   - Empty state div (class="state-empty"): icon/emoji + "Enter financial data and click Predict to see your risk assessment" text
   - Loading state div (class="state-loading"): skeleton elements (gauge skeleton, 3 text skeletons, 3 card skeletons)
   - Results state div (class="state-results"):
     - SVG gauge: viewBox="0 0 100 100", two circles (gauge-bg and gauge-fill, cx=50, cy=50, r=40, stroke-width=10, fill=none), gauge-bg stroke-dasharray="188.5 251.3", gauge-fill stroke-dasharray="0 251.3", text element for label. Risk category label below gauge
     - Probability display: "Risk Score: XX.X%" text
     - Debt payment defaulted note container (id="debtWarning", hidden by default)
     - Insights section: heading + container (id="insightsContainer") for dynamically inserted insight cards. Include summary paragraph container (id="insightsSummary")
     - Computed features section: heading + grid container (id="featuresGrid") for dynamically inserted feature items
   - Error state div (class="state-error"): error message container (id="errorMessage") + "Try Again" button (id="retryBtn")

4. **About the Model tab panel** (id="tab-about", class="tab-panel"):
   Static HTML content explaining:
   - Model architecture: "Input (9 features) -> 128 neurons (ReLU + Dropout) -> 64 neurons (ReLU + Dropout) -> Output (Sigmoid)"
   - The 9 features listed with brief descriptions: avg_income, avg_expenses, final_savings, debt_payment, credit_score, debt_ratio, liquidity_ratio, net_cash_flow, consec_negative_months
   - Stress definition: "Savings < 1 month expenses OR 3+ consecutive months negative cash flow"
   - Risk categories: Low (< 35), Medium (35-64), High (>= 65)
   - Model metrics container (id="modelMetrics") — placeholder for dynamic /health data (recall, ROC-AUC)
   - Training data: "Trained on 3,000 synthetic financial profiles across 4 archetypes"
   - Educational note about what the dashboard demonstrates

5. **Script tag:** `<script src="app.js"></script>` at end of body

Use semantic HTML elements where appropriate (main, section, nav, header). All IDs and classes must match exactly what styles.css targets and what app.js will query (per the patterns in 04-RESEARCH.md).
